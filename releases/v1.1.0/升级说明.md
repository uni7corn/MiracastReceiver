# Miracast Receiver v1.1.0 升级说明

## 版本概览

v1.1.0 重点增强 AirPlay 能力，新增基于 PhairPlay 实现思路的 iOS 屏幕镜像接收链路，并优化 Android TV 上的显示、画质和恢复体验。

## 主要更新

- 新增 iPhone / iPad AirPlay 屏幕镜像基础支持。
- 新增 AirPlay 2 配对、FairPlay 握手、NTP timing、镜像视频流接收和 H.264 硬解码链路。
- 新增 native FairPlay / ALAC 支持，包含 arm64-v8a 与 armeabi-v7a ABI。
- AirPlay 设备发现名称改为 TV 设备名称，与软件投屏显示名称保持一致。
- AirPlay 镜像分辨率根据 TV 显示模式自动协商，最高限制为 4K。
- 新增 AirPlay 镜像专用 Surface，避免 ExoPlayer 遮罩导致黑屏。
- 修复 iOS 竖屏镜像被强制拉伸的问题，改为等比例居中显示。
- 镜像播放时保持屏幕常亮，减少电视待机后 Surface 丢失导致无法恢复的问题。
- 移除重复 AirPlay mDNS 广播，避免 iOS 连接到错误的能力描述。
- README 增加 PhairPlay 项目参考说明。

## 安装包说明

本版本输出以下 APK：

- `miracast-receiver-v1.1.0-arm64-v8a-unsigned.apk`：适用于大多数现代 Android TV，推荐优先使用。
- `miracast-receiver-v1.1.0-armeabi-v7a-unsigned.apk`：适用于 32 位 Android TV 或较老设备。
- `miracast-receiver-v1.1.0-universal-unsigned.apk`：通用包，包含 32 位和 64 位 native 库，体积较大，适合不确定设备架构时使用。

> 注意：当前 release 包为 unsigned APK。如需直接安装到普通设备，请使用调试包或进行正式签名后安装。

## 升级建议

1. 建议先卸载旧版本或直接覆盖安装 v1.1.0。
2. 安装后重新打开应用，等待 AirPlay / DLNA / Miracast 服务启动完成。
3. iPhone 控制中心进入“屏幕镜像”，选择 TV 设备名称。
4. 如果 iPhone 缓存了旧设备名，建议关闭 Wi-Fi 后重新打开，或重启应用后再搜索。

## 已知限制

- AirPlay 屏幕镜像仍为持续优化中的基础实现，不同 iOS 版本和 Android TV 芯片可能存在兼容性差异。
- Windows 原生 Win+K 无线显示自动发现仍受 Android 普通应用权限限制。
- unsigned release APK 需要签名后才适合作为正式分发包。
